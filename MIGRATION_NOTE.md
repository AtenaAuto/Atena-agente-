# Migração de Estrutura do Repositório

Em 2026-04-23, a estrutura foi reorganizada para manter os arquivos do projeto na raiz do repositório,
em vez de uma pasta encapsulada (ex.: `ATENA_full/`).

Objetivo:
- facilitar clone/uso imediato;
- evitar nível extra de diretório;
- padronizar layout para CI/CD e deploy.
